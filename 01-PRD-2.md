## Executive Summary

**syntax.sh** is a terminal-based fiction writing and worldbuilding tool. It enables authors to write stories, manage characters, organize scenes, track plots, and understand their world—all in a beautiful, distraction-free terminal interface.

### Value Proposition

- **For Authors:** Distraction-free writing with built-in narrative planning tools
- **For Worldbuilders:** Organize locations, cultures, and timelines
- **For Storytellers:** Character database, plot tracking, scene management

### Target Users

1. **Primary:** Fiction writers (literary, sci-fi, fantasy, romance)
2. **Secondary:** Screenwriters, game writers
3. **Tertiary:** Journalists, bloggers, content creators

### Positioning

> "Your novel deserves a beautiful editor. syntax.sh is built for the story first, complexity second."

---

## Product Vision

### What It Is

A markdown-first writing environment with integrated story planning tools. Write stories, track characters, organize scenes, manage locations, plot your arc, get AI assistance, and export in multiple formats—all terminal-based.

### What It Is NOT

- Not a Scrivener replacement (more focused, less comprehensive)
- Not collaborative (single-user, local files)
- Not an AI writer (AI assists, human leads)

### Kyanite Suite Integration

**syntax.sh is standalone and independent but designed to integrate with future Kyanite tools:**
- Import color palettes from prism.sh (future)
- Export writing goals to focus.sh (future)
- Use as a reference in Kyanite documentation

## Core Features

### Feature 1: Split-Pane Editor

**Purpose:** Distraction-free, beautiful writing environment

**Core Mechanics:**
- Left pane: Markdown editor with live typing
- Right pane: Live markdown preview
- Toggle full-screen editing mode
- Real-time word/line count
- Syntax highlighting for markdown
- Auto-save every 30 seconds (non-blocking)
- YAML frontmatter for story metadata

**Technical Requirements:**
- Support 10,000+ line documents smoothly
- No lag during typing
- Preview renders within 100ms
- Auto-save is non-blocking

**Success Criteria:**
- ✅ Smooth typing at any document length
- ✅ Preview stays synchronized
- ✅ No data loss on auto-save
- ✅ Works at 80x24 terminal

**Status:** Core feature, MUST ship v1.0

---

### Feature 2: Character Database

**Purpose:** Organize and track characters

**Core Mechanics:**
- **Create:** Name, role, description, aliases
- **Track:** Age, occupation, personality, appearance, background
- **Relationships:** Links to other characters with tension levels
- **Character Arc:** Track character development
- **Quick Ref:** Lookup character while writing
- **Relationship Map:** ASCII diagram of connections
- **Search:** Find by name, role, or attribute
- **Export:** Character profiles to markdown

**Storage:**
- Location: `.story-name/characters/`
- Format: YAML metadata + markdown bio
- Auto-load when story opens

**Success Criteria:**
- ✅ Store 50+ characters without slowdown
- ✅ Search finds in <100ms
- ✅ Relationships track accurately
- ✅ Inline lookup while writing

**Status:** Core feature, MUST ship v1.0

---

### Feature 3: Scene Organization

**Purpose:** Structure story into hierarchical chapters and scenes

**Core Mechanics:**

- **Scene Outline:** View all scenes with summaries
- **Compile:** Assemble all scenes into single document

**Technical Requirements:**
- Handle 100+ scenes
- Navigate smoothly (<100ms)
- Preserve scroll position
- Deletion with confirmation

**Success Criteria:**
- ✅ Navigate 20+ scenes smoothly
- ✅ Outline shows full structure
- ✅ Compile creates valid document
- ✅ Scene metadata saved

**Status:** Core feature, MUST ship v1.0

### Feature 6: AI Story Assistant

**Purpose:** Non-intrusive AI feedback on story elements

**4 Quick Modes:**

1. **Character Check** (Ctrl+Shift+C)
   - Is character consistent?
   - Do actions align with personality?
   - Flag inconsistencies

2. **Dialogue Polish** (Ctrl+Shift+D)
   - Does dialogue sound natural?
   - Identify awkward exposition
   - Suggest improvements

3. **Scene Energy** (Ctrl+Shift+E)
   - Is scene paced well?
   - Is tension appropriate?
   - Suggest cuts or additions

4. **Plot Consistency** (Ctrl+Shift+P)
   - Does scene fit timeline?
   - Character location makes sense?
   - Flag plot holes

**Technical Requirements:**
- Use 3B model (lightweight)
- Response time <2 seconds
- Prompts under 200 tokens
- Graceful offline mode

**Success Criteria:**
- ✅ All 4 modes work
- ✅ Responses under 2 seconds
- ✅ Suggestions are actionable
- ✅ No hallucinations

**Status:** Core feature, MUST ship v1.0

## User Workflows

### Workflow 1: Starting New Story

```
1. syntax new "The Escape"
2. Create project
3. Add story metadata
4. Create main character
5. Create main location
6. Sketch outline
7. Write Chapter 1, Scene 1
Time: 10 minutes
```

---

### Workflow 2: Daily Writing Session

```
1. syntax open "The Escape"
2. Navigate to current scene
3. Resume writing
4. After scene, check scene energy
5. Make quick adjustments
6. Move to next scene
Time: 30-60 minutes
```

---

### Workflow 3: Managing Characters Mid-Story

```
1. While writing, need new character
2. Ctrl+Shift+N for quick create
3. Enter name, role, one-line
4. Return to writing
5. Fill details later
Time: 2 minutes
```

---

### Workflow 4: Checking Plot Consistency

```
1. Finish scene with big revelation
2. Ctrl+Shift+P for plot check
3. AI flags timeline issue
4. Review and adjust
5. Re-check
Time: 5 minutes
```

---

### Workflow 5: Compile & Export

```
1. Story 80% complete
2. Ctrl+E to export
3. Select "PDF for agent"
4. Auto-compile completed scenes
5. Format professionally
6. Save to file
Time: 1 minute
```
